# Contributing

Thanks for helping improve CacheSense.

## Before you open a PR

- Keep changes focused and small.
- Add or update examples when user-facing behavior changes.
- Run the test suite before submitting.
- Prefer wrappers and adapters that preserve existing cache workflows.

## Suggested workflow

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Run tests and example scripts.
5. Open a pull request with a short summary of the behavior change.
